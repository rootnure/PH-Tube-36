Q: Discuss the scope of var, let, and const

var⟹ Variable declared using var will be a global scope or function scope variable. Means, variables declared using var will be hoisted to the global root or the function root depending on the declaration position. var allow to redeclare same variable multiple times which may cause problem.
Note: Only the declaration part will be hoisted not the value.
For Example: If we write var name = "rootnure"; anywhere except function scope the variable name can be found anywhere form that script. No matter if it is declare inside a loop scope, condition scope or global scope. And if it declared inside a function it can only be found inside that function.

let⟹ Variable declared using let will be a block scope variable. Means, the variable only can be accessed inside that block only. Block can be function block, condition block, loop block.
For Example: If we write let age = 25; inside a block scope the variable age can only be found inside that specific block where it was declared.

const⟹ Scope for variables declared using const is same as let. It will also be a block scope variable. Here the block also can be function block, condition block, loop block.
For Example: If we write const weight = "82 KG"; inside a block scope the variable weight can only be found inside that specific block where it was declared.

Q: Tell us the use cases of null and undefined

null⟹ It means the value is not present right now and may got updated when available. It may also indicate that the data is not provided yet.
For Example: If we write let age = null; it means no age data is available for now. But may got updated when we get the DOB (Date of Birth) data.

undefined⟹ It means the value is not assigned yet. Only the declaration part may or may not assigned. It indicates variable is not declared or if declared no value is assigned.
For Example: If we write let name; it means no value is assigned but may use it in the future.

Q: What do you mean by REST API?

API means Application Programming Interface.
REST means Representational State Transfer architectural style.
A REST API is a specific type of API that follows the following guidelines:
  • Use JSON as the Format for Sending and Receiving Data.
  • Use Filtering, Sorting, and Pagination to Retrieve the Data Requested.
  • Use Nouns Instead of Verbs in Endpoints.
  • Use Status Codes in Error Handling.
  • Use Nesting on Endpoints to Show Relationships. etc.
When a client requests a resource using a REST API, the server transfers back the current state of the resource in a standardized representation.
